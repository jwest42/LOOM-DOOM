- **`StraightLineProjectile` base class** — `LaserScan`, `CircleBack`, `RecruiterFireball`, `WellnessPulse` all share an identical update-loop body (lifetime → newPos → player-hit → wall-check → assign). 5 instances of duplicated wall-bounds check (`iy < 0 || iy >= grid.length || ...`) — extract `isWallOrOutOfBounds(map, x, y)` helper.
- **`TrackingProjectile` base class** — `DocumentationRequest`, `TrackingDoc` share turn-toward-player + step + collision logic.
- **`RangedEnemy` base class** — `MiddleManager`, `HRSkeleton`, `SurveillanceDrone`, `WellnessOfficer` all share idle→sight→chase→attack-with-cooldown structure.
- **`MIN_CHASE_DISTANCE`** constant convention — currently each enemy uses bare-literal "stop closing" threshold (HRSkeleton 2.5, MiddleManager 2, VPOfSales 3, SurveillanceDrone 4). Hoist to named constants per file matching the existing `HEALTH`/`SPEED`/`SIGHT_RANGE` convention.
