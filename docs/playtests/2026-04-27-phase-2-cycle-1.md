## Bugs caught during Phase 2 implementation

Both surfaced when writing the Phase 2 e2e spec (Task 2.17) — the test infrastructure paid for itself immediately:

### Bug A: HUD never rendered (regression introduced in Task 2.5)
The state-machine refactor (commit `b69afe9`) accidentally removed the `loopReady` `useState` flag that gated `<LoomHud>` rendering, replacing it with `loopRef.current` in the JSX condition. Refs don't trigger re-renders, so the HUD was perpetually invisible after Task 2.5. Fixed by restoring the `loopReady` state and calling `setLoopReady(true)` after async map load.

### Bug B: WebGL `loseContext()` invalidates canvas across map transitions
Phase 1's final-review fix introduced `gl.getExtension('WEBGL_lose_context')?.loseContext()` in `Renderer.destroy()`. This **permanently invalidates** the canvas's WebGL2 context. Since `LOOMGame` reuses the same `<canvas>` element across map transitions (cyc1_lobby → cyc1_cubicles → cyc1_hr_arena), the second `Renderer` constructor always failed with `"shader compile failed: null"`. Fixed by removing the `loseContext()` call. The other resource cleanup (deleteShader/Program/Buffer/Texture) is still in place.

Both fixes shipped in commit `f6d2939` alongside the Phase 2 e2e spec.
