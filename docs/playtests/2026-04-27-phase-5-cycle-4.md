The implementation includes a fallback close-range damage path that activates when pointer lock fails, so the boss is still defeatable in test environments — but the visual fake-cursor-leaves-the-canvas effect ONLY shows in real browser sessions. **Manual playtest is the only way to verify the creative payoff of phase 3.**
